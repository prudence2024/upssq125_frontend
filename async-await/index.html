Javscript is single threaded, this means that javscript runs on a single thread and this means that

  • It can do one thing at a time
  • It has only one call stack
  • It processes code synchronously by default

This single threaded nature makes javscript easier to use especially in browsers, but it poses a challenge and this is the challeng, how do you handle asynchronous task like fetching data or waiting for a time line

Javascript execution system(runtime)

To handle both synchronus and asynchronous operations, javscript run times like v8 in chrome or node.js include several component

  • Call stack: this is where functions are executed line by line.
  • Web Apis / Node apis:Browser or node specific c++ apis that handles task like settimeout, fetch
  • Call back/task queue:task queue(callback queue/macro/queue):this store callbacks from web apis e.g ui events
  • Micro task queue:This store promises, queueMicroTask, and MutationObserver callbacks
  • Event loop:The event loop is the orchestrator, it checks if the call stack is empty, then pushes task from the queue into the stack. The event loop is the system that checks if the call stack is empty, if yes it runs all task in the Microtask queue then runs the first Microtask and finally it repeats forever

Loop order

  • Check if the callstack is empty
  • run all macro task
  • task one macro task from the task queue ad put it on the call stack
  • Execute it
  • Repeat
