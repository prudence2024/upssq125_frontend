Functional Programming and Object-Oriented Programmming

Functional Programming (Fp) and Object-Oriented Programmming (OOP) are 2 major programming paradigms each with a unique philosophy and approach to solving problems, javascript is a multi-paradigms meaning that it supports both fp and oop styles, understanding them help you write better and more maintainable code.

Functional Programming (Fp)

Functional Programming is a programmming paradigm (approach or pattern or model or framework) where programs are built using pure functions and the state(all the data stored up by variables) and data are immutable(unchanging). It emphasises declarative code where logic is expressed without describing control flow.

Core concept of in Fp

  1. Pure Function: A function is pure if it always return the same output for the same input, if it does not cause side-effect(no i/o- writing to the console , reading user input, writing/reading files or databases, sending network request.) (A side effect is any observable interaction with the outside world, outside world or mutation outside the function e.g modifying a global variable, making an HTTP request, writing to the console/DOM, changing the file system.) if it doesn't affect the global state .
  2. Immutablility: You don't modify existing data, you create and return new data.
  3. First Class and higher order function: In javascript, functions are treated as any other value, you can assign them to variables, pass them as argument to other functions(callback functions), return them from other functions, this is what we mean as functions being first class citizen.
  4. Higher Order functions: A higher order function is function that does at least one of the following. It takes one or more functions as argument, it returns a function,
  5. function composition: combination small function
  6. Avoid side effect

Benefit of Fp

  1. It is easier to test and debug
  2. It has predictable behaviour
  3. Encourage small reusable functions
  4. Easier concurrency(no shared mutable state)

OOP

OOP is a programming paradigms that uses objects to model real world entities.Each object contains state(data) and behaviour(methods). The goal is to organise code into reusable blue print (classes).

Core concept of in OOP

  1. Class: A class is a blue print. an object is an instance of a class
  2. Encapsulation:It hides internal state and only exposes a public interface.
  3. Inheritance: Objects can inherit properties from other object.
  4. Polymorphism: This is similar to inheritance but different
  5. Abstraction: It is about only the relevant features and hide the complex details

Benefit of Oop

  • Model(scaled down example of a system that has alll characteristics and features of a system, it is a representation of data and the logic that manages that data in your application.It is the blueprint or template for how your application understand and manipulates certain kind of informatioon eg user, product, blog post. ) real world problems

  • Reusability
  • It is easily extensible
