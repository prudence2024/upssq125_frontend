Css Media Query

The @media rule, introduced in CSS2 , made it possible to define different style rules for different media types.

Media queries in CSS3 extended the CSS2 media types idea: Instead of looking for a type of device they look at the capability of the device.

Media queries can be used to check many things such as:

  • Width and Height of the view port
  • Width and Height of the device
  • Orientation of the view port (Landscape or portrait).
  • Orientation of the device (Landscape or portrait).
  • Resolution of the device.

Using Media queries is a popular technique for delivering a tailored stylesheet to desktops, laptops, tablet, and mobile phones such as iphones and androids

CSS Media Types

Value Description
All Use for all media types
Print Used for Page materials and document viewed on the screen in print mode
Screen Used for Computer screens, Tablets, smartphones , etc.
Speech Used for Speech Synthesiazers

CSS Common Media Features

Feature Description
Orientation Orientation of the viewport (Landscape or portrait)
Max Height Maximum Height of the viewport
Min Height Minimum Height of the viewport
Min Width Maximum width of the viewport
Width Width of the viewport including the scroll bar
Height Height of the viewport including the scroll bar

Media Query Syntax

This are the set of rules taht guide how media queries are written

            
               @media not | only media-type and (media-feature) and (media-feature) {
                /*CSS rules */
               }

The media type is optional (if it is omitted it will be set to all) However, if you use not or only you must also specify the media type

The result of the query is true if the specified media type matches the type of device the document is being displayed on all media features in the media are true.When a media query is true, the corresponding stylesheet or style rules are applied following the normal cascading rules.
